# Barter 项目中文注释术语表

本文档定义了 Barter 项目中中文注释使用的术语规范，确保整个项目注释的术语一致性。

## 🔧 核心组件（保持英文）

这些是项目的核心组件名称，在注释中保持英文，必要时可添加中文说明。

| 英文               | 中文说明   | 使用示例                                      |
| ------------------ | ---------- | --------------------------------------------- |
| `Engine`           | 交易引擎   | Engine 是 Barter 的核心组件，负责处理交易事件 |
| `Strategy`         | 策略       | Strategy trait 定义了算法交易策略的接口       |
| `RiskManager`      | 风险管理器 | RiskManager 用于检查和管理交易风险            |

| `Position`         | 持仓       | Position 表示当前持有的交易头寸               |
| `Trade`            | 交易       | Trade 表示已执行的交易记录                    |
| `Balance`          | 余额       | Balance 表示账户资产余额                      |

## 🎯 使用规范

### 1. 核心组件名称

-   **规则**：在注释中保持英文，必要时添加中文说明
-   **示例**：
    -   ✅ "Engine 是 Barter 的核心组件"
    -   ✅ "Strategy trait 定义了算法交易策略的接口"
    -   ❌ "引擎是 Barter 的核心组件"（应使用 Engine）

### 2. 概念性术语

-   **规则**：使用中文翻译
-   **示例**：
    -   ✅ "算法交易系统"
    -   ✅ "回测功能"
    -   ❌ "Algorithmic Trading 系统"（应使用中文）

### 3. 技术术语

-   **规则**：根据上下文选择，技术说明用英文，通俗解释用中文
-   **示例**：
    -   ✅ "这个 Trait 定义了接口"（技术说明）
    -   ✅ "这个接口定义了..."（通俗解释）

### 4. 混合格式

-   **规则**：首次出现时使用"英文（中文）"格式，后续可直接使用中文
-   **示例**：
    -   ✅ "Engine（引擎）负责处理交易事件。引擎会..."
    -   ✅ "MarketEvent（市场事件）包含..."

## 📝 注释示例

### 好的注释示例

````rust
/// Engine（引擎）是 Barter 的核心组件，负责处理算法交易逻辑。
///
/// Engine 会：
/// - 处理输入的市场事件和账户事件
/// - 维护内部状态（持仓、订单等）
/// - 根据 Strategy 生成算法订单
/// - 通过 RiskManager 检查订单风险
///
/// # 使用示例
///
/// ```rust,ignore
/// let mut engine = Engine::new(...);
/// engine.process(market_event)?;
/// ```
pub struct Engine {
    // ...
}
````

### 不好的注释示例

```rust
/// 引擎是 Barter 的核心组件
///
/// 引擎会处理事件
pub struct Engine {
    // ...
}
```

**问题**：

-   应使用 "Engine" 而非 "引擎"
-   缺少详细说明
-   缺少使用示例
